# 管理端测试版部署指南 (Port 8081 外网)

本文档说明如何将 `user_admin_web` 部署到 Nginx，并通过端口转发实现外网访问。

## 1. 端口映射说明

- **外网访问端口**: 8081
- **内网 Nginx 端口**: 8080
- **转发规则**: 外网 8081 -> 内网 8080

## 2. 项目构建

在 `user_admin_web` 目录下执行构建命令：

```bash

cd user_admin_web

npm install

npm run build:test

```

构建完成后，生成的静态文件位于 `user_admin_web/dist` 目录。

## 3. Nginx 配置

测试环境使用 `user_admin_web/nginx_test.conf`。主要配置如下：

- **监听端口**: 8080
- **根目录**: `/var/www/html/admin_test`
- **后端代理**: `http://127.0.0.1:8000`

### 配置文件位置

建议将 `nginx_test.conf` 复制到 Nginx 的配置目录（如 `/etc/nginx/sites-enabled/admin_test`）。

## 4. 部署步骤

1. **创建部署目录**:

   ```bash

   sudo mkdir -p /var/www/html/admin_test

   ```

2. **同步文件**:

   将构建好的 `dist` 内容复制到部署目录：

   ```bash

   sudo cp -r dist/* /var/www/html/admin_test/

   ```

3. **应用 Nginx 配置**:

   ```bash

   sudo cp nginx_test.conf /etc/nginx/sites-enabled/admin_test

   sudo nginx -t

   sudo systemctl reload nginx

   ```

## 5. 访问地址

部署完成后，可以通过以下地址访问：

`http://服务器外网IP:8081`

## 6. 注意事项

- 确保服务器防火墙已放行 8080 端口。
- 测试环境的后端 API 地址默认为 `http://127.0.0.1:8000`，请确保后端服务已启动。

# 后端服务部署与打包总结

本文档总结了 QASystem 后端服务的部署与打包流程。

## 1. 技术栈

- **语言**: Python 3.11+
- **框架**: FastAPI
- **包管理工具**: [uv](https://docs.astral.sh/uv/)
- **数据库**: MySQL (使用 Alembic 进行迁移管理)
- **AI/RAG 技术栈**: LangChain, Chroma, Ollama (本地部署), BGE-M3 (向量化模型), Qwen2.5 (大语言模型)

## 2. 环境准备

- **Python**: 确保已安装 Python 3.11 或更高版本。
- **uv**: 推荐使用 `uv` 进行依赖管理。
- **MySQL**: 数据库需提前运行，并根据 `.env` 配置好连接。
- **Ollama**: 本地运行 Ollama，并拉取所需的模型 (`bge-m3`, `qwen2.5`)。

## 3. 安装与配置

### 3.1 克隆并进入目录

```bash
cd server
```

### 3.2 配置文件

根据 `.env.example` 创建 `.env` 文件，并根据实际环境修改以下关键变量：

- `DATABASE_URL`: 数据库连接字符串。
- `SECRET_KEY`: JWT 签名密钥。
- `PDF_PATH` / `CHROMA_PERSIST_DIRECTORY`: 数据存储路径。

### 3.3 安装依赖

使用 `uv` 安装依赖并创建虚拟环境：

```bash
uv sync
```

## 4. 数据库管理

使用 Alembic 执行数据库迁移：

```bash
uv run alembic upgrade head
```

## 5. 打包与启动

### 5.1 本地/开发运行

```bash
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 5.2 生产环境部署 (后台运行)

可以使用 `nohup` 或系统服务 (如 Systemd) 来管理进程。

```bash
nohup uv run uvicorn app.main:app --host 0.0.0.0 --port 8000 > server.log 2>&1 &
```

## 6. 与前端集成

前端 (User Client / User Admin Web) 通常由 Nginx 托管。Nginx 配置中应包含对 `/api/` 路径的反向代理：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

## 7. 日志管理

运行日志通常记录在 `server/logs/` 目录或启动脚本指定的日志文件中。
