## 功能

### 只有指定的用户才能审核通过文档

在前端界面定义一个可以审核的用户用户名列表
只有在这个列表中才能审核通过文档
通过设置一个函数，只有在列表中的才能渲染审核界面

### 编辑知识库文档


1. 数据库升级：在 document 表中新增了 source_data 字段（LONGTEXT 类型），用于存储生成文档时的原始 JSON 数据。
2. 后端服务更新：
       * 修改了 KnowledgeTemplateService，在生成文档时同步将原始数据保存到数据库。
       * 新增了 GET /api/v1/rag/document/{filename}/content 接口，用于根据文件名获取“源码”数据。

3. 前端管理端更新：
       * API 层：在 api/rag.ts 中添加了 getDocumentSourceData 方法。
       * 列表页 (KnowledgeView.vue)：在已发布文档的操作列中新增了“编辑”按钮，点击后跳转至编辑器并携带 edit 参数。
       * 编辑器 (KnowledgeTemplateForm.vue)：
           * 支持检测路由中的 edit 参数。
           * 自动从后端拉取原始数据并回填表单（实现回显）。
           * 在编辑模式下禁用标题修改（保持文件名一致以实现覆盖更新）。
           * 动态调整提交按钮文字为“更新知识库文档”。

  注意事项：
   * 历史数据：由于此字段是新添加的，在本次修改之前生成的旧文档没有
     source_data，因此点击编辑会提示“加载失败”或数据不存在。新生成的文档将完美支持再次编辑。
   * 覆盖逻辑：编辑提交后，后端会重新生成同名的 .docx 和 .pdf 文件并覆盖旧文件，状态会重置为
     pending（待审核），需要管理员重新审核后才能正式入库更新向量库。
