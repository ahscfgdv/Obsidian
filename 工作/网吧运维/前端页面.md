## 1. 技术栈

- **框架**: Vue 3 (Composition API)
- **构建工具**: Vite
- **UI 组件库**: Element Plus
- **路由管理**: Vue Router 4
- **状态存储**: LocalStorage (用于持久化 Token)
- **Markdown 渲染**: markdown-it (用于 AI 回复展示)
## 2. 目录结构

- `src/layout/`: 包含全局布局组件，如 `MainLayout.vue`（侧边栏、导航等）。
- `src/views/`: 业务页面组件。
    - `LoginView.vue`: 登录页面。
    - `ChatView.vue`: AI 问答对话界面。
    - `TicketView.vue`: 工单提交界面。
- `src/router/`: 路由配置及导航守卫。
- `public/`: 静态资源。
## 3. 核心功能实现

### 身份验证 (Authentication)

- **登录**: 登录成功后，后端返回的 JWT Token 会存储在 `localStorage.getItem('token')` 中。
- **路由守卫**: 在 `src/router/index.js` 中定义了全局前置守卫：
    - 访问 `requiresAuth: true` 的路由（如 `/` 和 `/ticket`）时，若无 Token 则重定向至 `/login`。
    - 已登录状态下访问 `/login` 会自动跳转至首页。
- **退出登录**: 清除 `localStorage` 中的 Token 并跳转至登录页。
### AI 问答界面 (`ChatView.vue`)

- 实现了一个典型的对话式 UI。
- 使用 `markdown-it` 解析 AI 返回的内容，支持代码块和格式化显示。
- 集成了对话历史记录（由后端持久化，前端负责展示）。

### 统一布局 (`MainLayout.vue`)

- 使用 Element Plus 的 `el-container` 容器。
- 侧边栏集成了路由跳转功能，并实时高亮当前激活菜单。

## 4. 开发与构建

在 `web_admin` 目录下运行：

- **安装依赖**: `npm install`
- **本地开发**: `npm run dev`
- **生产构建**: `npm run build`
- **预览产物**: `npm run preview`

## 5. 配置与 API 调用

- 前端通过 `fetch` 或 `axios`（通常配置在全局或具体组件中）调用后端接口。
- 默认后端 API 基址通常对应 `http://localhost:8000/api/v1`。
