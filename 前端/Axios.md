在现代前端开发中，虽然浏览器原生自带了 `fetch` API，但 **Axios** 依然是开发者们的首选。这主要归功于它对 HTTP 请求进行了极佳的封装，提供了一系列“开箱即用”的高级特性。

以下是 Axios 的核心优点：

---

### 1. 强大的拦截器（Interceptors）

这是 Axios 最受欢迎的特性。你可以定义“请求拦截器”和“响应拦截器”：

- **请求拦截：** 在请求发送前自动注入 `Token`、显示全屏 Loading 或修改请求头。
    
- **响应拦截：** 在数据到达组件前统一处理错误码（如 401 自动跳转登录）、简化数据结构（如直接返回 `res.data`）。
    

### 2. 自动转换 JSON 数据

- **原生 fetch：** 你需要手动调用 `res.json()` 才能拿到对象数据。
    
- **Axios：** 它会自动检测响应头，如果返回的是 JSON，它会自动解析成 JavaScript 对象，省去了繁琐的转换步骤。
    

### 3. 更广泛的浏览器兼容性

- Axios 是基于经典的 `XMLHttpRequest` (XHR) 构建的，甚至可以兼容老旧浏览器（如 IE）。
    
- 相比之下，`fetch` 是现代 API，在一些旧版本环境中需要引入额外的 Polyfill 才能运行。
    

### 4. 客户端支持防止 CSRF

Axios 内置了简单的防御机制，它可以自动从 Cookie 中读取 Token 并将其添加到请求头中，帮助开发者抵御跨站请求伪造（CSRF）攻击。

### 5. 支持请求取消（Cancel Token）

在一些场景下（例如用户快速切换页面，或者搜索框连续输入），旧的请求已经没有意义。Axios 允许你通过 `AbortController` 或 `CancelToken` 轻松取消还在进行中的请求，节省带宽和资源。

### 6. 更好的错误处理

- **fetch 的痛点：** 只有网络故障时才会抛出错误。如果服务器返回 `404` 或 `500`，`fetch` 依然认为请求成功（`ok` 为 false）。
    
- **Axios 的优点：** 只要状态码不在 `2xx` 范围内，Axios 就会自动进入 `.catch()` 流程，让错误捕获逻辑更符合直觉。
    

### 7. 同构性（Node.js & 浏览器通用）

Axios 既可以在浏览器端运行（基于 XHR），也可以在 Node.js 环境下运行（基于原生的 `http` 模块）。这意味着你可以用同一套代码编写前后端的网络请求逻辑。

---

### 对比总结表

|**特性**|**Axios**|**原生 Fetch**|
|---|---|---|
|**拦截器**|✅ 内置支持|❌ 需自己封装|
|**JSON 自动转换**|✅ 自动完成|❌ 需手动 `.json()`|
|**请求取消**|✅ 简单易用|✅ 支持 (AbortController)|
|**超时设置 (Timeout)**|✅ 属性配置|❌ 需配合 `setTimeout`|
|**CSRF 防御**|✅ 内置支持|❌ 不支持|
|**错误触发**|状态码非 2xx 即报错|仅网络断开才报错|

---

### 实际建议

虽然 `fetch` 越来越好用，但在**企业级项目**中，**Axios** 依然是更好的选择。因为它提供的拦截器和错误处理机制能帮你减少大量的重复代码。
